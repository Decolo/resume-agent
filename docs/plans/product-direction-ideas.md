# Resume Agent 产品方向建议

## 1. 从"工具"到"工作流"
- 当前：用户给 prompt → agent 执行，依赖用户自己组织意图
- 建议：固化为引导式阶段流程
  - 分析现有简历（结构、内容、格式评估）
  - 理解目标岗位（用户输入或抓取 JD）
  - 差距分析（简历 vs 岗位要求的 gap）
  - 针对性改写（措辞、经验突出、关键词优化）
  - 格式输出（选择目标格式导出）
- 价值：降低使用门槛，非技术用户也能用

## 2. JD 匹配能力
- 用户贴 JD 链接或文本，agent 自动做匹配度分析
- 输出：匹配度评分 + 具体改进建议 + 自动改写
- 解决核心痛点："这份简历投这个岗位行不行"
- 已有 `web_fetch`/`web_read` 基础设施可复用

## 3. 质量反馈闭环
- 当前缺失：agent 改完简历，用户无法判断"改得好不好"
- 建议增加评审环节：
  - Recruiter 视角打分（格式规范、信息完整度、表达专业度）
  - ATS 兼容性检查（关键词密度、格式是否可解析）
  - 改进前后量化对比（具体列出改了什么、为什么改）
- 价值：给用户信心，形成"修改→评估→再优化"的闭环

## 4. 行业模板与知识库
- 不同行业简历写法差异大：
  - SWE：项目经验、技术栈、量化成果
  - PM：产品指标、跨团队协作、商业影响
  - 设计：作品集链接、设计流程、工具链
  - 金融：交易规模、合规经验、证书资质
- 内置行业最佳实践 + 模板，agent 输出质量会有质的提升
- 可作为差异化竞争力

## 5. 多交付形态
- Web UI：最大化用户群体（优先）
- API Service：让第三方集成（招聘平台、求职工具）
- VS Code 插件：留在开发者生态（低优先级，CLI 已覆盖）
- 选择取决于目标用户定位
